PR title: Add ingest scaffold, strict schemas, validator and Notion/schema quick-check workflow

Summary
- Adds JSON schemas for intake, provenance, and requirements output.
- Adds an ingest pipeline scaffold: `tools/ingest_pipeline.py` (discover → validate → emit; dry-run + audit).
- Reworked `tools/validate_provenance.py` to perform JSON Schema validation with a CLI.
- Adds unit/smoke tests:
  - `tests/test_schemas.py`
  - `tests/test_ingest_pipeline.py`
  - `tests/test_validate_provenance.py`
  - `tests/test_notion_dryrun.py`
- Adds a lightweight GitHub Actions workflow `.github/workflows/notion-schema-check.yml` that runs the schema/validator tests and Notion dry-run on PRs (no secrets required).
- Adds a README badge for the quick-check workflow.
- Removes a temporary artifact `tmp_notion_table.md`.

Why
- Provide early validation of intake files and provenance using JSON Schemas.
- Add a small ingest scaffold and tests so we can iterate safely.
- Add a fast PR check (Notion dry-run + schema tests) to catch regressions early.
